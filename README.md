# 월간 계획 캘린더

Firebase Realtime Database와 연동된 개인 플래너 웹앱입니다.
별도 서버 없이 `index.html`을 브라우저에서 바로 열어 사용할 수 있습니다.

---

## 프로젝트 구조

```
my_calendar/
├── index.html          # HTML 구조
├── style.css           # 전체 스타일
└── js/
    ├── utils.js        # 상수, 공유 상태, 날짜·색상 유틸
    ├── firebase.js     # Firebase 초기화, 데이터 저장/동기화
    ├── holidays.js     # 공휴일 로드·파싱·표시
    ├── calendar.js     # 월간 캘린더 렌더, 일정 추가/수정/삭제
    ├── week.js         # 주간 캘린더 렌더
    └── detail.js       # 상세 패널, 세부일정 CRUD, 복사
```

> 스크립트 로딩 순서: `utils.js` → `firebase.js` → `holidays.js` → `calendar.js` → `week.js` → `detail.js`

## 주요 기능

### 월간 캘린더
- 현재 월 기준 전체 날짜 표시
- 오늘 날짜 하이라이트
- 일/토요일, 공휴일 색상 구분
- 날짜 셀 클릭 → 일정 추가 모달

### 주간 캘린더
- 이번 주 7일 컬럼 레이아웃
- 이전/다음 주, 이번 주 이동 버튼
- 월간과 동일 데이터 공유

### 일정 관리
- **일정 추가**: 텍스트, 색상 선택, 시작일~종료일 지정 가능
- **기간 일정**: 시작일~종료일 설정 시 해당 날짜에 스팬 바로 표시
- **수정·삭제**: 인라인 수정, ✕ 버튼으로 삭제
- **복사**: 다른 날짜로 일정 복사 (세부일정 완료 상태는 초기화)
- **CSV 내보내기**: 현재 월 일정 CSV 다운로드

### 진행률 표시
- 세부일정이 있는 항목은 이름 옆에 `(완료/전체)` 형태로 표시
- 달력 아이템 색상이 완료 비율에 따라 원래 색 → 보라색으로 그라데이션

### 세부일정 (상세 패널)
- 항목 클릭 시 우측 패널 오픈
- 세부일정 추가·완료 체크·수정·삭제·순서 변경
- 완료 시 완료 날짜 자동 기록 (클릭하여 수정 가능)
- 완료된 세부일정은 완료 날짜 셀에도 표시

### 항목 완료 토글
- 세부일정이 없는 항목은 상세 패널에서 **✅ 완료** 버튼으로 완료 처리
- 완료 시 달력에서 보라색으로 표시, **↩️ 되돌리기**로 취소 가능

### 공휴일
- **URL 가져오기**: JSON 또는 iCal URL 입력 후 자동 파싱
  - 예) `https://date.nager.at/api/v3/PublicHolidays/2026/KR`
- **직접 붙여넣기**: `.ics` 파일 내용 붙여넣기
- 공휴일명이 달력 셀과 주간 컬럼 헤더에 표시
- 데이터는 `localStorage`에 저장 (새로고침 후에도 유지)
- CORS 차단 시 에러 메시지에 브라우저 직접 열기 링크 안내

## Firebase 연동

1. [Firebase 콘솔](https://console.firebase.google.com)에서 프로젝트 생성
2. **Realtime Database** 활성화 (테스트 모드)
3. 앱 시작 화면에서 `firebaseConfig` 객체 전체를 붙여넣고 **저장 및 연결** 클릭
4. 이후에는 자동으로 연결 (설정은 `localStorage`에 저장됨)

## 사용 기술

- Vanilla JS (ES6+), 번들러 없음
- Firebase Realtime Database v9 (compat)
- iCal / JSON 파싱 (직접 구현)
- localStorage (공휴일, Firebase 설정 캐시)
